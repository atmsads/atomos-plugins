# exec-content-marketing 에이전트 작업 계획

> ION 콘텐츠 마케팅 팀장용 Claude Code 플러그인. 본 문서는 **에이전트 제작 착수 전 설계도** 역할이며, 구현이 시작되면 체크리스트에 따라 실제 파일을 채워 나간다.

## 1. 목적 / 배경

- 2026-04-15 콘텐츠 마케팅 팀장(현승님)과의 미팅에서 5개 업무 중 **스토리보드/대본 기획**이 자동화 1순위로 확정됨.
- 팀 전체 영상 제작량: **5명 × 일 3개 × 베리에이션 = 일 45개.** 대본 기획이 매일의 병목.
- 현승님 본인은 실무를 하지 않음. 하지만 암묵지(소구점/후킹/논리 도출)의 밀도가 가장 높은 사람.
- 현승님이 이전에 Claude로 만들어둔 대본 기획 프로젝트가 있었으나, **갱신 부재로 성과가 떨어진 상태.** 본 에이전트는 이를 보편화·재갱신 가능한 형태로 재구축하는 것이 목표.

## 2. 사용자 & 운영 흐름

```
현승(콘텐츠 마케팅 팀장, 파일럿) ──▶ 민지(실무 컨펌자, 이관 타깃)
         │
         └─ 현승님이 먼저 써서 "이거 쓸 만하다" 검증되면 민지 님에게 설치·교육 후 이관.
            피드백은 현승님이 1차 수집 → 다음 턴 암묵지 업데이트에 반영.
```

- 최종 생산 주체: 콘텐츠 마케터 5명 (민지 님이 2차 컨펌).
- 현승님 역할: 소구점 도출 주관(직접 머리로), 방향성 잡기, 팀장 간 원칙 정렬.

**③ 후킹 + 논리 선정 단계 — AI 자동 (2026-04-24 Q3 확정)**

**③-a 후킹 3개 선정** (소구도출.md §3초 후킹 10패턴에서 다수 생성·검토 후 3개 최종 선정)
- **선정 3기준** (`knowledge/03-소구-도출-철학.md`):
  1. **다양한 자극 메커니즘** — "사람의 뇌가 크게 3가지 형태로 소비심리가 구분됨 — 자극/지배/균형". 3개 후킹이 서로 다른 심리 버튼을 눌러야.
  2. **타겟 일관성** ★ — "타겟이 바뀔때마다 머신러닝이 꼬이고.." 후킹 다양성 ≠ 타겟 다양성. 3개 모두 동일 타겟에게.
  3. **3요소 통합** — "맛 + 맛 + 맛 보다 맛 + 신뢰 + 가격". 3초 내 USP 3요소 교차 가능한 후킹만.
- **패턴 풀**: pain point / BnA / 결과형 / 숫자 / 경각심 / 성적 / 리얼후기 / 대세형 / 만족형 / 공감형 / 키워드
- **약한 소구 배제**는 기준 아닌 전제 필터.
  - ⚠️ **"최신" 레퍼런스가 아니라 "검증된" 레퍼런스** — 현승님 원칙.
- **화자 변수**(사투리/애기/대표/고객 리뷰)는 AI가 후킹별로 자동 결정.

**③-b 논리 포맷 2개 선정**
- `knowledge/03-소구-도출-철학.md` §6단계 구조(후킹 → 라포르 → 권위 → 주장 → 근거 → 사례) 기반 변형 2개:
  - **변형 α (공감·신뢰형)**: 후킹 → 라포르 → 권위 → 주장 → 사례 — 고관여 제품 (건기식·전자제품·고가)
  - **변형 β (속도·결과형)**: 후킹 → 주장 → 근거 → 사례 → CTA — 저관여·충동구매 (식품·소모품·프로모션)
- **기본은 α·β 둘 다 채택** (구조가 뚜렷하게 달라 A/B 테스트 가치 높음).
- 제품 관여도가 한쪽으로 압도적이면 **같은 계열 내 세부 변형 2개** (예: α 내에서 "라포르 우선" vs "권위 우선").

**④ 대본 단계 — AI 자동, 정확히 5개 (2026-04-24 Q3 확정)**

> ⚠️ **분리 후보 (v1 실측 후 재평가, 2026-04-25)**: ④는 향후 `script-writer` subagent 로 분리 가능성 있음. 분리 이득 — (1) 리서치 raw·knowledge 전체가 카피 작성 컨텍스트를 오염하지 않도록 격리, (2) 5개 대본 **병렬 호출**로 후반부 품질 저하 방지, (3) `brand-researcher`/`competitor-landscape` 분리 사상과 일관. 분리 시 메인은 타겟·소구점·후킹·논리를 **고정값으로 명시 전달**하여 머신러닝 꼬임 방지 원칙 유지. **v1 은 통합으로 구현 → 실측 후 병목이 ④로 확인되면 분리.** brand-researcher v0→v1 교정 경로와 동일 패턴.
- ③에서 선정된 **후킹 3 × 논리 2 조합으로 대본 5개 생성** (2×2 A/B 테스트 + 탐색 1개):

| # | 후킹 | 논리 | 역할 |
|---|---|---|---|
| 1 | 후킹A (가장 강한 소구) | 변형α | A × 공감·신뢰형 |
| 2 | 후킹A | 변형β | A × 속도·결과형 |
| 3 | 후킹B (다른 자극 메커니즘) | 변형α | B × 공감·신뢰형 |
| 4 | 후킹B | 변형β | B × 속도·결과형 |
| 5 | 후킹C (보완·신규 각도) | α or β 택1 | 탐색 슬롯 |

- **조합 의미**: 후킹A·B는 2×2 매트릭스로 **후킹 vs 논리 어느 쪽이 전환에 기여하는지 분리 측정**. 후킹C는 예상 밖 각도 실험.
- 각 대본 구조: **후킹 멘트 → 본문 → CTA.**
- **분량 기준 (knowledge/03 원문, 2026-04-26 정정)**: "30초 안에 설득" + "5초 안에 약 3문장" → 30초 영상 ≈ **약 18문장**. 후킹은 첫 3초 (knowledge/04 §후킹 1번). 임의 임계값(예: "6~10문장") 추가 금지.
- **서브텍스트 (연극 대본식 — 기본 활성, 2026-04-26 변경)**: 각 라인에 **지문(화자 액션·표정·톤) + 카메라 지시(샷 사이즈·B-roll·컷)** 함께 표기. 화자 속내 의도는 미표기 (영상 광고 대본 표준 범위). 카메라 약어: CU/ECU/MS/WS/POV/B-roll. 표기 규약: `[카메라]` `(지문)` `"대사"`.
- **출력 직전 self-QA**: `knowledge/04-영상-콘텐츠-체크리스트.md` §후킹 3원칙 + §내용 3원칙 (디자인 섹션은 영상 편집 영역이라 scope 밖, 적용 제외).

### 3.3 출력 포맷

**(A) 선택 근거 블록 — 최종 대본 앞에 필수 노출**

```
[이번 회차 선택 근거]
- 소구점: "{선택된 소구점}"  ← 사용자가 고른 것
- 타겟: "{확정된 타겟}" (5개 대본 전체 공통 — 머신러닝 일관성)
- 매체: "{선택된 매체}"
- 후킹 A: "{문장}" (패턴: {pain point 등}, 자극 메커니즘: {자극/지배/균형}, 선정 근거 1줄)
- 후킹 B: "{문장}" (패턴, 자극 메커니즘, 선정 근거 1줄)
- 후킹 C: "{문장}" (패턴, 자극 메커니즘, 선정 근거 1줄)
- 논리 α: {구조} — 선정 근거 1줄
- 논리 β: {구조} — 선정 근거 1줄

[대본별 조합 근거]
- 대본 #1 — 후킹A × 논리α: 조합 이유 1줄
- 대본 #2 — 후킹A × 논리β: ...
- 대본 #3 — 후킹B × 논리α: ...
- 대본 #4 — 후킹B × 논리β: ...
- 대본 #5 — 후킹C × {α or β}: ...
```

**(B) 대본 5개 — 복사·붙여넣기 가능한 형태 (2026-04-26 포맷 변경)**

구조 라벨("후킹 멘트:", "(라포르)", "(권위)" 등) 표기하지 않는다. 화자 1줄 + 카메라·지문·대사 라인이 위에서 아래로 흐르는 촬영 대본 형태. 단계 정보는 (A) 선택 근거 블록에서만 노출.

```
[대본 #1 — 후킹A × 논리α]
화자: {화자 변수}

[ECU: ...] (지문) "대사"
[MS: ...] (지문) "대사"
[B-roll: ...] (지문) "대사"
...
[WS: ... + 자막 "..."] (지문) "CTA 대사"
```

**(C) 피드백 루프 — 모든 대본 출력 후 필수 질문**

```
→ "이 대본 마음에 드세요?"
   1. 네, 마음에 들어요 (종료)
   2. 소구점부터 다시
   3. 후킹부터 다시
   4. 논리 포맷부터 다시
   5. 대본 말투·톤만 수정
   6. 직접 입력 (자유 텍스트)
```

재생성 로직:
- 2 → ②부터 재실행 (사용자 소구점 재선택)
- 3 → ③부터 재실행 (소구점 유지)
- 4 → ④부터 재실행 (소구점·후킹 유지)
- 5 → ④만 재실행하되 말투 지시 반영
- 6 → AI가 자유 입력 파싱 → 해당 단계 자동 재진입

피드백 루프는 사용자가 "1" 또는 만족 표시할 때까지 반복.

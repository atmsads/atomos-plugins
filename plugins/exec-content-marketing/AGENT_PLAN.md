# exec-content-marketing 에이전트 작업 계획

> ION 콘텐츠 마케팅 팀장용 Claude Code 플러그인. 본 문서는 **에이전트 제작 착수 전 설계도** 역할이며, 구현이 시작되면 체크리스트에 따라 실제 파일을 채워 나간다.

## 1. 목적 / 배경

- 2026-04-15 콘텐츠 마케팅 팀장(현승님)과의 미팅에서 5개 업무 중 **스토리보드/대본 기획**이 자동화 1순위로 확정됨.
- 팀 전체 영상 제작량: **5명 × 일 3개 × 베리에이션 = 일 45개.** 대본 기획이 매일의 병목.
- 현승님 본인은 실무를 하지 않음. 하지만 암묵지(소구점/후킹/논리 도출)의 밀도가 가장 높은 사람.
- 현승님이 이전에 Claude로 만들어둔 대본 기획 프로젝트가 있었으나, **갱신 부재로 성과가 떨어진 상태.** 본 에이전트는 이를 보편화·재갱신 가능한 형태로 재구축하는 것이 목표.

## 2. 사용자 & 운영 흐름

```
현승(콘텐츠 마케팅 팀장, 파일럿) ──▶ 민지(실무 컨펌자, 이관 타깃)
         │
         └─ 현승님이 먼저 써서 "이거 쓸 만하다" 검증되면 민지 님에게 설치·교육 후 이관.
            피드백은 현승님이 1차 수집 → 다음 턴 암묵지 업데이트에 반영.
```

- 최종 생산 주체: 콘텐츠 마케터 5명 (민지 님이 2차 컨펌).
- 현승님 역할: 소구점 도출 주관(직접 머리로), 방향성 잡기, 팀장 간 원칙 정렬.

## 3. 에이전트 스펙: `storyboard-writer`

### 3.1 입력 — 대화형 순차 질문 (2026-04-22 확정)

`/draft-storyboard` 호출 시 에이전트가 **하나씩 순차 질문**. 인라인 인자 파싱 안 함 (실무자 정보 누락 방지).

| 순서 | 항목 | 필수 | 기본값 |
|---|---|---|---|
| 1 | 브랜드/업체명 | O | — |
| 2 | 제품명 | O | — |
| 3 | 매체 | X | 인스타 (Meta/Instagram) |

- 퍼널 단계 입력 제거됨 (불필요한 인터럽트).
- 추가 컨텍스트는 피드백 루프에서 자연어로 수용.

### 3.2 4단계 파이프라인

```
[1] 리서치  →  [2] 소구점 (3+1)  →  [3] 후킹 5개  →  [4] 대본 3~5개
  ✅ 확정       사용자 선택 후          AI 자동              최종 산출물
```

**사용자 개입은 소구점 선택 한 번만.** ③·④는 AI 자동 진행 → 최종 출력에서 "선택 근거" 노출 + 피드백 루프로 교정.

**① 리서치 단계 — ✅ 완료 (v1 배포, 2026-04-22)**

구현 파일: `agents/brand-researcher.md` (v1)
검증 상태: 1차 실측(5케이스, 2026-04-20) + 2차 검증(3케이스 병렬, 2026-04-22) + 실운영 테스트(감자밭 고구마빵, 2026-04-22) 모두 통과.

**아키텍처: Claude only, 외부 API 0개.**
- 도구: `WebSearch` + `WebFetch` + **Playwright MCP** (이미 설치됨, JS 렌더링/iframe/스크린샷) + **Task tool subagent**
- 패턴: 옵션 (c) 변형 — `brand-researcher` **subagent** 분리 (2-step)
  - subagent에 리서치 컨텍스트(수십 페이지 본문) 격리 → 메인 대본 생성 컨텍스트 오염 방지
  - Anthropic 멀티 에이전트 Research 시스템과 동일 사상

**v1 주요 룰** (v0 → v1 2차 실측 교정 반영):
- Google 쿼리 2~3개 연속 0건 시 네이버 SERP(`search.naver.com`) Playwright 조기 전환 — 한국 스몰브랜드 대응
- 네이버 블로그는 모바일 URL(`m.blog.naver.com`) 1차, PC+iframe fallback — 스니펫 견고성 확보
- SPA 쇼핑몰(무신사/글로우픽/29CM/지그재그 등) → WebFetch 생략, Playwright 직행
- Cafe24 자사몰 한글 슬러그 404 우회 → `?product_no=N` 직접 URL
- 식품(F&B) 카테고리 소스 분기 행 추가 (자사몰/스마트스토어/쿠팡/맘카페)
- 패션·뷰티는 NEW/EXCLUSIVE/BEST 주력 SKU 상세 페이지 WebFetch 필수
- 편향 경고: 공식몰 베스트 리뷰·협찬 블로거 과반 시 "체리피킹 위험" 플래그 출력
- 리서치 에이전트는 법적·광고 심의 판단 하지 않음 (메인 에이전트 영역)

**폐기된 아이디어** (명시적): 매체(인스타) 특화 정탐 절차 — 로그인 벽으로 공개 정보 극히 제한적, brand-researcher는 매체 중립 유지. 인스타 톤앤매너는 현승님 암묵지 + 사용자 직접 입력 슬롯 영역.

**결정 근거 (대안 비교 결과)**
- 외부 Deep Research(GPT/Gemini/Perplexity) MD 선주입 안 씀 — 외부 의존성 + 갱신 부담
- Tavily/Exa/네이버 API MCP 안 씀 — 외부 키 관리 부담, 팀장 직접 운영에 부적합
- 같은 LLM이어도 결과가 다른 진짜 원인은 (1) 검색 인프라, (2) 도구 사용 RL 학습, (3) 스캐폴딩 — 이 중 (3)은 우리가 메울 수 있음

**커버리지 현실 (인지된 한계)**

| 소스 | 접근 가능 | 비고 |
|---|---|---|
| 공식 사이트 / 뉴스 / 위키 / 글로벌 | ✅ | WebFetch로 충분 |
| 네이버 뉴스 | ✅ | URL 알면 읽힘 |
| 네이버 블로그 / 무신사 / 다나와 | ✅ | Playwright MCP로 JS 렌더링 |
| 인스타 공개 프로필 | △ | 부분, 로그인 벽 |
| 카페 / 인스타 댓글 / 유튜브 댓글 | ❌ | 모든 AI 공통 한계 |

→ 남은 갭은 **검색 인덱스 깊이 한 가지** (Anthropic backend가 Google보다 한국 웹 얕다고 추정). fan-out 쿼리(같은 주제 5~10개 표현으로 분할 검색)로 보정.
→ 소비자 인사이트(카페·리뷰 톤)는 **현승님 암묵지 + 사용자 직접 입력 슬롯**으로 메움 (애초에 사람이 더 잘 아는 영역).

**선행 검증 (구현 직전)**
- [ ] **실측**: 현승님이 자주 다루는 한국 D2C 브랜드 1개로 (WebSearch + WebFetch + Playwright) 조합이 어디까지 뽑는지 30분~1시간 확인
- 이 결과로 subagent 프롬프트 깊이/fan-out 쿼리 수 결정

**② 소구점 단계 (3+1 옵션)**
- AI가 제품 × 타겟(퍼널 단계 반영) × 매체 교차점 기반으로 소구점 **후보 3개 제시**.
- 각 후보에 근거(본질적 욕망·페인포인트·시장 통념) 1줄 첨부.
- **4번째 옵션: 사용자 직접 입력** 자유 텍스트 슬롯 (AI 후보 모두 불만족 시).
- 암묵지 주입 지점: 현승님 소구점 도출 문서(예정).

**③ 후킹 단계 — AI 자동**
- 선택된 소구점에 대해 AI가 후킹 5개를 내부적으로 생성·검토 후 **1개 선정**.
- 카테고리별 검증된 레퍼런스 패턴 기반 (빅볼피터 / 페인포인트 직접 / 구글프터러 / 신뢰 코팅 등).
  - ⚠️ **"최신" 레퍼런스가 아니라 "검증된" 레퍼런스** — 현승님 원칙.
- 화자 변수(사투리/애기/대표/고객 리뷰)는 AI가 소구점·제품 컨텍스트에 맞춰 자동 결정.
- 암묵지 주입 지점: 카테고리별 후킹 패턴 문서(예정).

**④ 대본 단계 — AI 자동**
- 선정된 후킹에 대해 논리 포맷 3~5종 조합해 대본 3~5개 생성.
- 논리 포맷 후보: 주장 → 권위(실내 자료) → 라포 → 주장 → 사례 → 리뷰 / 뷰티 B4F 등.
- 구조: **후킹 멘트 → 본문 → CTA.**
- 서브텍스트(연극 대본식): 기본 비활성, 플래그로 on.
- 암묵지 주입 지점: 카테고리별 논리 포맷 문서(예정).

### 3.3 출력 포맷

**(A) 선택 근거 블록 — 최종 대본 앞에 필수 노출**

```
[이번 대본의 선택 근거]
- 소구점: "{선택된 소구점}"  ← 사용자가 고른 것
- 후킹: "{AI 선정 후킹}" (패턴: 빅볼피터 / 페인포인트 직접 등)
  이유: (왜 이 후킹을 골랐는지 1~2줄)
- 논리 포맷: "{선정 포맷}"
  이유: (왜 이 포맷인지 1~2줄)
- 화자: "{화자 변수}"
```

**(B) 대본 3~5개 — 복사·붙여넣기 가능한 형태**

```
[대본 #1 — 논리 포맷: 주장→권위→라포]
후킹 멘트: ...
본문: ...
CTA: ...
```

**(C) 피드백 루프 — 모든 대본 출력 후 필수 질문**

```
→ "이 대본 마음에 드세요?"
   1. 네, 마음에 들어요 (종료)
   2. 소구점부터 다시
   3. 후킹부터 다시
   4. 논리 포맷부터 다시
   5. 대본 말투·톤만 수정
   6. 직접 입력 (자유 텍스트)
```

재생성 로직:
- 2 → ②부터 재실행 (사용자 소구점 재선택)
- 3 → ③부터 재실행 (소구점 유지)
- 4 → ④부터 재실행 (소구점·후킹 유지)
- 5 → ④만 재실행하되 말투 지시 반영
- 6 → AI가 자유 입력 파싱 → 해당 단계 자동 재진입

피드백 루프는 사용자가 "1" 또는 만족 표시할 때까지 반복.

### 3.4 핵심 원칙

- **"대본만 좋다고 광고 효율 나오는 건 아니다."** 영상 대본은 제품·랜딩·소재 3꼭짓점 중 **한 꼭짓점**일 뿐. 에이전트 프롬프트에 명시적으로 경고 문구 포함.
- `exec-base` 공통 원칙 준수 (한국어, "값+기간+출처" 3종 세트, 모호함 1회 재질문).

## 5. 플러그인 등록 (예정)

`.claude-plugin/marketplace.json` 의 `plugins` 배열에 다음 엔트리 추가:

```json
{
  "name": "exec-content-marketing",
  "source": "./plugins/exec-content-marketing",
  "description": "콘텐츠 마케팅 팀장용 환경 — 영상 광고 스토리보드·대본 기획 자동화",
  "version": "0.1.0",
  "category": "executive",
  "tags": ["content", "marketing", "storyboard", "script", "executive"]
}
```

설치 경로: `/plugin install exec-content-marketing@ion-execs`

## 6. 범위 제외 (의도적)

- **context-graph 연동**: `exec-base/hooks/log-sync.sh` 가 placeholder 상태. 팀장님 주관 트랙이며 본 작업과 별개.
- **영상 편집 포인트 에이전트**: 회의에서 후순위로 정리.
- **촬영 기획안 초안**: 빈도 낮음(난이도 5/빈도 2), 이번 주기엔 제외.
- **디자인 팀장용 플러그인 `exec-design`**: 본 플러그인 완성 후 별도 진행.

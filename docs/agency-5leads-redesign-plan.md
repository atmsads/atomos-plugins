# 마케팅 에이전시 5팀장 업무 재설계 계획 (ion-execs)

> 대상: 마케팅 전략 · 광고 운영 · 디자인 · 개발 · 영상 제작 팀장 5인 · 장소: 오프라인 120분 · 결과물: 팀별 plugin 설계 초안 + 다음 파일럿 2팀 선정
> 전제 현실: 현재 `ion-execs` marketplace 에는 `exec-base` + `exec-marketing` **2개**만 존재. `exec-marketing` 은 ATMS MCP 기반으로 광고 운영 팀장용으로 가동 중. 나머지 4팀(전략/디자인/개발/영상)은 **미구현 — 이번 회의에서 설계 착수**.
> 에이전시 컨텍스트: 외부 클라이언트 다수 · 캠페인 단위 딜리버리 · 크리에이티브(디자인+영상) 와 집행(광고 운영) 이 분리된 분업 구조.

---

## 1. 회의 목적 (3줄)

- 광고 운영 팀장(`exec-marketing`) 파일럿 결과를 공유하고, **크리에이티브 딜리버리 라인(전략 → 디자인/영상 → 개발 → 광고 운영)** 전체를 Claude Code plugin 으로 묶을 그림을 확정한다.
- 각 팀장의 주간 반복 업무를 4분류(자동화/준자동화/보조/비적용)로 매핑하고, **팀장 승인이 실제 딜리버리 병목인 지점**을 찾아낸다.
- 다음 파일럿 팀 **2개**(딜리버리 체인 상 연결되는 조합 권장: 예 전략+광고 운영 / 디자인+영상) 를 이 회의에서 결정한다.

---

## 2. 회의 전 준비 (각 팀장 지참물)

공통:
- 본인의 **1주일 반복 업무 리스트 10개** (빈도 · 소요시간 · 사용 툴)
- 최근 1개월 **클라이언트 요청 샘플 3건** (어디서 받음 → 어떻게 가공 → 누구에게 전달)
- 팀장 본인이 **의사결정 병목**이라고 인지하는 Top 3 업무
- 외부에 나가면 안 되는 **민감 지표/산출물** 목록

팀별 추가 지참물:

| 팀장 | 추가 지참물 |
|---|---|
| 마케팅 전략 | 최근 3건 신규 제안서 · 월간 리포트 샘플 1부 · 경쟁사 모니터링 대시보드 URL |
| 광고 운영 | (이미 운영 중) 지난 주 `/daily-brief` 출력 1부 · 개선 요청 사항 |
| 디자인 | 주간 크리에이티브 요청 큐 캡처 · Canva 팀 URL · 브랜드 가이드 PDF |
| 개발 | 운영 중 랜딩페이지 URL 리스트 · 픽셀/GTM 계정 구조 · 이슈 트래커 링크 |
| 영상 제작 | 진행 중 촬영 스케줄 · 편집 대기 영상 리스트 · 영상 소재 성과 레퍼런스 1건 |

> 가정: 전원 Claude Code CLI 설치 완료 · `/plugin install exec-base@ion-execs` 수행 상태 · 현재 환경에 붙어있는 MCP: **ATMS / Notion / Slack / Gmail / Google Calendar / Canva / Context7**.

## 3. 회의 어젠다 (120분)

| 시간 | 세션 | 산출 |
|---|---|---|
| 0–10 | 배경 & ion-execs 구조 · 에이전시 딜리버리 라인과 plugin 매핑 설명 | 공통 이해 |
| 10–25 | `exec-marketing` 라이브 시연: `/daily-brief`, `/diagnose-campaign`, `ads-ops` | 벤치마크 |
| 25–50 | 팀별 업무 리스트 낭독 + 4분류 워크시트 개별 작성 (섹션 5) | 팀별 매핑표 초안 |
| 50–65 | 크로스팀 병목 매핑: "내 승인 대기가 다른 팀을 얼마나 멈춰 세우는가" (섹션 6) | 병목 맵 |
| 65–90 | plugin 설계 산출물 채우기 — agent/command 후보 명세 (섹션 7) | 팀별 초안 스펙 |
| 90–105 | 다음 파일럿 팀 2개 선정 (섹션 8 스코어링) | 우선순위 결정 |
| 105–120 | 성공 지표 · 리스크 · 2주 후속 액션 (섹션 9–11) | 체크리스트 |

## 4. 에이전시 딜리버리 라인과 plugin 매핑 (개요)

```
  [클라이언트 브리프]
        │
        ▼
  마케팅 전략 팀장 ──(전략/제안서/KPI)──▶
        │
        ├─▶ 디자인 팀장 ──(광고 소재: 정지/배너)──┐
        │        └──(웹사이트 시안/UI)──▶ 개발 팀장 ──▶
        │                                            ├─▶ 광고 운영 팀장 ──(집행/성과)──▶ [클라이언트 리포트]
        ├─▶ 영상 제작 팀장 ──(영상 소재)─────────────┘
        │
        └─▶ 개발 팀장 ──(랜딩페이지·트래킹·브랜드 사이트 구현)──▶ (광고 운영 연결)
```

- plugin 경계: 위 박스 = `plugins/exec-<team>/` 1개.
- 공통 규약(`exec-base`): 출력 포맷(`exec-briefing`), 수치 3종세트(값+기간+출처 MCP tool), 외부 상태 변경 자동 실행 금지, 스코프 차단 훅.
- **크로스팀 산출물**(예: 전략 → 디자인 요청, 디자인 → 광고 운영 소재 넘김) 은 Notion/Slack 을 공용 버스로 사용. plugin 은 "자기 팀 업무에서 이 버스에 쓰고/읽는" 역할만 한다.

## 5. 4분류 워크시트 (팀별 템플릿 · 예시 포함)

각 팀장은 지참한 업무를 아래 4칸에 매핑. **판정 기준은 `exec-redesign-playbook.md §4` 와 동일.** 에이전시 특화 예시를 팀별로 제시한다.

### 5-1. 마케팅 전략 팀장 — `exec-strategy` (미구현, 이번에 설계)

| 분류 | 예시 업무 | 근거 MCP tool (가정 포함) |
|---|---|---|
| A. 자동화 | 월간 클라이언트 성과 리포트 초안 (ATMS 성과 + 전월 대비) | `company_ad_performance_summary`, `company_roi_summary` + Notion 템플릿 |
| A. 자동화 | 어제 전 클라이언트 핵심 지표 다이제스트 | `company_ad_performance_summary` (여러 company 순회) |
| B. 준자동화 | 신규 제안서 초안 (업종/예산/KPI 입력 → 전략+매체믹스 초안) | ATMS 과거 동종업 레퍼런스 + Context7(프레임워크 참고) + Notion 제안서 템플릿 |
| B. 준자동화 | 경쟁사 크리에이티브 트렌드 요약 | **가정**: 외부 광고 라이브러리 MCP 미구현 → 수동 URL 입력 기반 |
| C. 보조 | 클라이언트 미팅 전 회사 히스토리 훑기 | Notion `notion-search` + Slack 채널 히스토리 |
| D. 비적용 | 클라이언트 협상 · 신규 수주 피칭 · 단가 합의 | — |

### 5-2. 광고 운영 팀장 — `exec-marketing` (구현 완료, 확장 영역)

| 분류 | 예시 업무 | 근거 MCP tool |
|---|---|---|
| A. 자동화 | `/daily-brief` — 어제 전 계정 성과 | `company_ad_performance_summary`, `company_roi_summary` (구현됨) |
| A. 자동화 | `/weekly-rollup` (신규) — 주간 매체별 성과 + 예산 소진률 | `campaign_performance_summary` 다건 + `autorule_execution_insights` |
| B. 준자동화 | `/diagnose-campaign <id>` — 원인 후보 + 권장 조치 | `campaign_diagnosis`, `adset_diagnosis` (구현됨) |
| B. 준자동화 | `/budget-rebalance` (신규) — 목표 ROAS 기준 예산 재배분안 | `company_roi_summary` + `adset_performance_summary` — **집행 자동 실행 금지** |
| C. 보조 | `ads-ops` 에이전트 대화형 탐색 (구현됨) | ATMS 전반 |
| D. 비적용 | 매체 파트너 미팅 · 신규 매체 계약 | — |

### 5-3. 디자인 팀장 — `exec-design` (미구현, 이번에 설계)

> 디자인팀은 **광고 소재(배너/썸네일/정지/영상 프레임)** 와 **클라이언트 웹사이트 디자인(랜딩페이지·브랜드 사이트 UI/UX)** 두 축을 모두 담당. 두 축은 요청 경로·산출물·품질 기준이 달라 큐와 commands 를 분리한다.

| 분류 | 예시 업무 | 축 | 근거 MCP tool (가정 포함) |
|---|---|---|---|
| A. 자동화 | 주간 디자인 요청 큐 요약(광고 소재 / 웹사이트 분리) | 공통 | Notion `notion-query-database-view` (가정: 두 축 DB 분리 또는 필터 태그) |
| A. 자동화 | 팀원별 작업량 분포 (광고 소재 vs 웹 프로젝트 공수 분해) | 공통 | Notion 요청 DB + 담당자/태그 필드 |
| B. 준자동화 | 소재 피로도 체크 → 재제작 후보 리스트 | 광고 소재 | ATMS `ad_performance_summary` (CTR 하락) + Canva 원본 매핑 |
| B. 준자동화 | 신규 광고 요청 → 기존 템플릿 매칭 제안 | 광고 소재 | Canva `authenticate` 후 리소스 조회 (**가정**: 템플릿 검색 tool 추가 필요) |
| B. 준자동화 | 웹사이트 프로젝트 진행률 · 단계별 지연 리스크 리포트 | 웹사이트 | Notion 프로젝트 DB(단계: 기획/와이어/시안/QA/배포) + Google Calendar(마일스톤) |
| B. 준자동화 | 웹사이트 시안 → 브랜드 가이드 준수 점검(컬러/폰트/로고·컴포넌트 일관성) | 웹사이트 | 이미지/URL 입력 + Claude 시각 검토 + Notion 가이드 참조 |
| B. 준자동화 | 라이브 랜딩/브랜드 사이트 반응형·접근성(a11y) 기본 점검 | 웹사이트 | **가정**: 크롤링/Lighthouse MCP 미구현 → Context7 로 가이드 인용 + 수동 URL 입력 |
| C. 보조 | 광고 소재 브랜드 가이드 준수 점검 | 광고 소재 | 수동 체크리스트 + Claude 이미지 검토 |
| C. 보조 | 웹사이트 레퍼런스·트렌드 수집 (업종/무드별) | 웹사이트 | 수동 URL + 요약 |
| D. 비적용 | 디자이너 1:1 피드백 · 포트폴리오 리뷰 · 클라이언트 크리에이티브 디렉션 | 공통 | — |

> 축 간 경계 주의: 웹사이트 프로젝트는 **개발팀 딜리버리와 맞물림**(시안 → 퍼블/개발 → 배포). `exec-design` 과 `exec-dev` 의 공용 Notion DB 스키마를 섹션 11 Week 2 에서 확정한다. 광고 소재 큐는 광고 운영팀 론칭 일정과 맞물리므로 `exec-marketing` 쪽에서 읽을 수 있는 필드(캠페인 ID, 론칭 예정일) 를 필수로 둔다.

## 6. 크로스팀 병목 맵 (에이전시 전용 세션)

각 팀장이 1줄씩 기입:

- "내가 (팀장이) 승인을 늦게 주면 → (어느 팀) 의 (어떤 업무)가 멈춘다" × 3개
- "내가 기다리는 입력 중 가장 자주 늦는 것은 → (어느 팀) 의 (어떤 산출물)" × 3개

세션 목표: **에이전시 딜리버리 라인 상 팀장 승인이 실제 막히는 엣지** 를 3개 이상 찾아 plugin 의 `/need-approval` 류 커맨드로 일괄 꺼내는 설계로 바꾼다.

예상 패턴:
- 전략 → 디자인/영상: 제안 확정 → 크리에이티브 요청 전달 지연
- 디자인/영상 → 광고 운영: 소재 승인 ↔ 집행 론칭 지연
- 개발 → 광고 운영: 랜딩/트래킹 확인 ↔ 캠페인 ON 지연

## 7. plugin 설계 산출물 (팀별 1장 · 회의 종료 시 제출)

각 팀장은 아래를 채운다. 포맷 레퍼런스: `plugins/exec-marketing/`.

### 7-1. 공통 제출 항목

1. **Agent 명세 1–2개** (`name` / `description` + 프로토콜 본문)
2. **Slash command 3개** (일일 다이제스트형 1 / 단건 진단형 1 / 주간·월간 롤업형 1)
3. **MCP 서버 의존성** — 현재 붙어 있는 것 vs **가정: 신규 필요** 분리

## 8. 우선순위 선정 규칙 (다음 파일럿 2팀)

3축 1–5점 스코어링 + 에이전시 딜리버리 라인 보너스.

| 축 | 1점 | 5점 |
|---|---|---|
| 반복 빈도 (주간) | 월 1회 이하 | 매일 |
| 의사결정 비용 | 10분/영향 미미 | 1시간+/클라이언트 만족·매출 직결 |
| MCP 연결 난이도 (역점수) | 신규 MCP 개발 필요 | 이미 붙어있음 (ATMS/Notion/Slack/Gmail/GCal/Canva/Context7) |

**에이전시 보너스**
- 광고 운영(`exec-marketing`) 과 **인접한 팀**(전략/디자인/영상) 은 +1점. 딜리버리 라인이 붙어 있어 공동 활용 시너지가 크다.
- 개발팀은 크롤링/GTM MCP 미구현 이슈가 있어 현실적으로는 Wave 2 권장.

**동점 처리**
- 되돌리기 비용 낮은 팀 우선.
- 팀장 본인 프로세스가 고정화된 팀 우선(고정화 안 된 팀은 plugin 화 효과가 반감).

**필수 탈락**
- 데이터 소스가 오프라인/엑셀 수작업 전용
- MCP "가정"이 전부이고, 담당자 없이 외부 API 연동 필요

## 9. 성공 지표 (KPI · 파일럿 후 4주 측정)

- **팀장 주간 절감 시간**: ≥ 3시간/주 (에이전시 특성상 클라이언트 리포트 비중 고려)
- **클라이언트 리포트 리드타임**: 요청 → 전달 가능 상태 ≤ 30분 (이전 기준선 기록 필수)
- **팀장 호출 빈도**: plugin command ≥ 10회/주
- **수치 신뢰도**: 값+기간+출처(MCP tool명) 3종세트 누락 0%
- **크로스팀 승인 리드타임**: 승인 대기 → 처리까지 평균 시간(섹션 6 기반) 50% 단축
- **재작업률**(디자인/영상): plugin 도입 전 대비 재작업 요청 비율 -20%

> 측정: 팀장 자가 리포트 + `SessionEnd` 훅의 `log-sync.sh` 활용. **가정**: `EXEC_LOG_TARGET` 환경변수로 Notion DB 또는 S3 로 transcript pipe — 현재 스크립트 placeholder, 파일럿 시작 전 구현 필요.

## 10. 리스크 · 거버넌스 (에이전시 특화)

- **클라이언트 데이터 혼선**: 여러 클라이언트를 동시에 다루므로, `scope-check.sh` 에 **클라이언트 단위 워크스페이스 분리 룰** 반드시 추가. 타 클라이언트 지표가 다른 클라이언트 출력에 섞이면 계약 위반. exit code `2` + stderr 로 사유 명시.
- **집행 변경 금지**: 광고 ON/OFF · 예산 증감 · 랜딩페이지 배포 · 소재 교체 · 메일 발송 모두 **plugin 자동 실행 금지**. 제안·초안까지만. (`ads-ops` 원칙 전사 격상.)
- **크리에이티브 기밀**: 미공개 캠페인 소재(영상/디자인) 링크가 agent 로그에 남지 않도록 agent 지시문에서 명시적으로 제외.
- **출처 3종세트 강제**: 수치 = 값 + 기간 + 출처(MCP tool명). 지키지 않는 agent/command 출력은 기본 차단.
- **팀장 간 데이터 경계**: 광고 운영 팀장이 클라이언트 원가/수수료를 못 보게, 디자인/영상 팀장이 ATMS 수익성 전체를 못 보게 — 팀별 ATMS 권한 범위 회의에서 확정.
- **MCP 자격증명**: 팀장 개인 `settings.json` 에서 관리. 팀장 교체 시 토큰 회수 + plugin 재설치 프로세스 필요.
- **감사 로그**: `SessionEnd` 훅은 실패해도 세션 방해 금지(exit code 무관). 감사용 pipe 스크립트는 **가정: 미구현 → Week 1 내 작성**.
